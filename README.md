# Google Trends Skill for Uzum Sellers

> AI-скилл для Claude Code, который даёт селлерам Uzum Marketplace доступ к данным Google Trends прямо из терминала — анализ трендов, сравнение товарных категорий, мониторинг сезонности и поиск растущих ниш для принятия решений по ассортименту.

---

## Проблема

Селлеры на маркетплейсах принимают решения об ассортименте **вслепую**:

| Проблема | Последствия |
|----------|------------|
| Не знают, какие товары сейчас в тренде | Упущенная прибыль на хайповых товарах |
| Не понимают сезонность спроса | Закупка товара, который не продаётся в этом сезоне |
| Не могут сравнить категории между собой | Инвестиции в менее прибыльную нишу |
| Не отслеживают конкурентов | Потеря доли рынка |
| Анализ требует ручной работы в браузере | Часы вместо секунд |

**Результат:** По данным исследований, до 40% новых товарных позиций на маркетплейсах не окупаются из-за неправильного выбора ниши и тайминга.

---

## Решение

**Google Trends Skill** — это AI-скилл для [Claude Code](https://claude.ai/claude-code), который превращает AI-ассистента в аналитика рыночных трендов. Селлер общается с ассистентом на естественном языке и получает данные Google Trends без ручного поиска.

### Как это работает

```
Селлер: "Какие товары сейчас в тренде в Узбекистане?"

AI-ассистент (с Google Trends Skill):
├── Получает RSS-фид трендов Google для UZ
├── Парсит и анализирует топ-20 трендов
├── Фильтрует товарные/коммерческие тренды
└── Выдаёт рекомендации для ассортимента
```

### Установка — одна команда

```bash
npx skills add AliNek09/openclaw-google-trends
```

После установки скилл автоматически активируется в Claude Code. Никакой настройки не нужно.

## Возможности

### 1. Ежедневный мониторинг трендов

Получение актуальных трендов по любой стране за секунды.

```
"Покажи сегодняшние тренды в Узбекистане"
"Что сейчас ищут в России?"
"Какие мировые тренды сегодня?"
```

**Поддерживаемые страны:** UZ (Узбекистан), RU (Россия), US (США), DE (Германия), TR (Турция), CN (Китай), KR (Корея) и 50+ других.

### 2. Сравнение товарных категорий

Прямое сравнение интереса к нескольким товарам/категориям для выбора наиболее перспективной ниши.

```
"Сравни интерес к кроссовкам, кедам и сандалиям в Узбекистане"
"Что популярнее: iPhone или Samsung в Центральной Азии?"
"Сравни спрос на детскую одежду vs детские игрушки"
```

### 3. Анализ сезонности

Понимание циклических паттернов спроса для планирования закупок.

```
"Покажи сезонность запроса 'кондиционер' за последние 3 года"
"Когда пик спроса на школьные принадлежности?"
"Как менялся интерес к пуховикам по месяцам?"
```

### 4. Поиск связанных запросов

Обнаружение смежных ниш и новых возможностей через анализ связанных поисковых запросов.

```
"Какие запросы связаны с 'фитнес' в Узбекистане?"
"Что ещё ищут люди, которые ищут 'умные часы'?"
```

### 5. Региональный анализ

Определение, в каких регионах товар наиболее востребован.

```
"Где больше всего ищут 'электросамокат'?"
"Сравни интерес к 'органическая косметика' по странам СНГ"
```

## Сценарии использования для селлеров Uzum

### Утренний анализ рынка

```
Селлер: "Дай мне утренний обзор трендов для Узбекистана и России"

Результат:
- Топ-20 трендов UZ за сегодня
- Топ-20 трендов RU за сегодня
- Выделены коммерчески значимые тренды
- Рекомендации: какие категории стоит рассмотреть
```

### Выбор новой ниши

```
Селлер: "Я хочу добавить новую категорию товаров. Сравни:
         бытовую технику, спортивные товары и товары для дома"

Результат:
- График интереса по каждой категории
- Тренд роста/падения
- Сезонные пики
- Рекомендация по приоритету
```

### Мониторинг конкурентов

```
Селлер: "Отслеживай бренды Xiaomi, Samsung и Apple в Узбекистане.
         Покажи, когда какой-то из них начинает расти"

Результат:
- Сравнительная динамика брендов
- Моменты резкого роста (новые релизы, акции)
- Возможности для таргетирования
```

### Автоматический ежедневный отчёт (Cron)

Настройка автоматического мониторинга через cron-задачу:

```javascript
{
  "name": "Ежедневный отчёт по трендам для Uzum-селлера",
  "schedule": { "kind": "cron", "expr": "0 9 * * *" },
  "payload": {
    "kind": "agentTurn",
    "message": "Получи тренды Google для UZ и RU. Выдели топ-10 коммерческих трендов. Сравни с вчерашними. Отметь новые тренды."
  }
}
```

## Примеры реальных инсайтов

### Пример 1: Обнаружение тренда

```
Запрос: "Тренды Узбекистана сегодня"
Обнаружено: Резкий рост запросов "робот-пылесос"

→ Действие: Добавить робот-пылесосы в ассортимент Uzum
→ Результат: Опережение конкурентов на 2-3 дня
```

### Пример 2: Сезонный анализ

```
Запрос: "Сезонность 'школьная форма' в Узбекистане"
Обнаружено: Пик спроса — июль-август

→ Действие: Начать закупку в июне, запустить рекламу в июле
→ Результат: Максимальные продажи в пиковый период
```

### Пример 3: Сравнение ниш

```
Запрос: "Сравни: yoga mat, гантели, скакалка"
Обнаружено: "yoga mat" растёт +40% за год, остальные стагнируют

→ Действие: Фокус на товарах для йоги
→ Результат: Инвестиция в растущую нишу
```

## Ограничения и дальнейшее развитие

### Текущие ограничения

- Google Trends предоставляет **относительные**, а не абсолютные данные
- Нет официального API (используются RSS и публичные endpoints)
- Данные отражают поисковый интерес, а не прямые продажи

### Roadmap

- [ ] Интеграция с Uzum Seller API для сопоставления трендов с реальными продажами
- [ ] Уведомления в Telegram при обнаружении резкого роста тренда
- [ ] Кастомные отчёты с визуализацией
- [ ] Поддержка Uzum-специфичных категорий товаров
- [ ] Мультиязычная поддержка (узбекский, русский, английский)
